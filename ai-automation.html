비개발자 운영 담당자로서 AI를 도구로 활용해 제본 서비스의 반복 수작업을 직접 자동화 시스템으로 전환했습니다. 단순 효율화에 그치지 않고 운영 데이터를 의사결정 자산으로 전환하고, 사람 개입 없이 돌아가는 무인 인프라와 표준 운영 체계까지 설계해 전 과정을 주도했습니다.

운영 데이터를 수집·정제·분석해 시즌·채널·고객 단위 인사이트로 전환.
리스크·연속성 관리
속인화된 지식을 SOP로 문서화, 검수 규칙 시스템화로 품질 편차·담당자 의존도 제거.

Project Details
1
POD 제본 주문서 자동화 시스템
문제·배경
매일 주문 DB를 수기 조회해 시트에 옮기고 제본용 PDF를 일일이 내려받던 반복 업무가 시간 소모·휴먼에러의 원인.
나의 역할
업무 전 과정을 단계별로 분해해 자동화 흐름을 설계, 비개발자도 운영 가능한 형태로 구현 (1_DB조회.bat, 2_PDF다운로드.bat 더블클릭 실행).
AI·도구
AI 협업 Python 스크립트 설계·구현, Windows 배치 파일, DB 직접 조회, 스프레드시트 연동.
성과
주문 접수→시트 정리→PDF 다운로드 수작업 파이프라인을 클릭 한 번으로 자동 실행, 코드를 몰라도 안정 운영하도록 패키징.
전략기획 관점 — 핵심 운영 프로세스의 병목을 식별하고, 빈도 높은 반복 업무부터 시스템화해 즉각적 생산성 확보.
2
무인 자동화 인프라 구축 (GitHub Actions + Self-hosted Runner)
문제·배경
사람이 직접 켜야만 돌아가던 자동화를 정해진 시간에 사람 개입 없이 실행되도록 만들 필요.
나의 역할
회사 PC를 self-hosted runner로 연결, 평일 정시마다 시트가 자동 업데이트되는 스케줄 파이프라인 구성.
AI·도구
GitHub Actions(CI/CD 스케줄링), self-hosted runner, 워크플로우 설계, Git/GitHub 버전 관리.
성과
평일 1일 5회 자동 시트 업데이트 무인 실행. ‘사람이 켜야 도는’ 자동화를 ‘알아서 도는’ 인프라로 격상.
전략기획 관점 — 일회성 효율화를 반복 가능한 운영 레버리지로 전환, 인력 투입 없이 처리량이 유지·확장되는 구조 설계.
3
매출 리포팅 봇 v2 (DB → Slack 자동 발송)
문제·배경
매출 현황을 매번 사람이 집계·공유하던 비효율과 보고 지연 제거.
나의 역할
DB 추출→가공→Slack 자동 게시 데이터 파이프라인 기획·구현. 평일 정시 자동 발송 + 특정 날짜 수동 재발송 옵션 설계.
AI·도구
DB 직접 쿼리, GitHub Actions 스케줄, Slack 연동, workflow_dispatch 수동 트리거.
성과
매일 오전 매출 리포트 채널 자동 게시로 집계·공유 시간 0으로.
전략기획 관점 — 경영 가시성 확보. 의사결정 지표를 정시·정형으로 자동 전달해 조직의 데이터 활용 습관 형성.
4
매출 데이터 분석 및 인사이트 도출
문제·배경
자동 수집된 주문/매출 데이터를 단순 보고를 넘어 의사결정에 활용할 필요.
나의 역할
성수기 시즌 패턴, B2B/B2C 채널 트렌드, 핵심 고객(학원) 데이터를 분석해 세그먼트로 매핑. 열람출력 세그먼트는 약 4개월간(’26.2~6) 130건·642만 원 매출을 직접 집계·정제(중복 주문 제외 포함).
AI·도구
데이터 분석, 채널/고객 세그멘테이션, 스프레드시트 집계, 중복·이상치 정제.
성과
반복 운영 데이터를 자산화해 시즌·채널·주요 거래처 단위 매출 패턴 파악 기반 마련.
전략기획 관점 — 매출 데이터를 전략 인풋으로 전환, 세그먼트별 기여도를 근거로 자원 배분·타깃 우선순위 논의 토대 구축.
5
비즈니스 규칙 자동 적용 (제본 사양 검수 자동화)
문제·배경
무선/와이어 제본의 페이지 수 조건 등 특정 사양은 사람이 매번 눈으로 확인해 안내를 표기해야 했음.
나의 역할
제본 방식·페이지 수 조건에 따라 신규 주문에 ‘안내필요’를 자동 표기하는 규칙 로직 정의·구현.
AI·도구
조건 기반 규칙 엔진, 데이터 자동 가공.
성과
검수 누락 위험이 큰 케이스를 시스템이 자동 식별·표기, 품질 편차 제거.
전략기획 관점 — 암묵적 운영 노하우를 명시적 비즈니스 룰로 표준화, 담당자가 바뀌어도 동일 품질 유지.
6
와이어 제본 표지 자동 생성 통합
문제·배경
와이어 제본 주문마다 표지를 별도로 만들던 수작업.
나의 역할
PDF 다운로드 자동화 프로세스 안에 표지 자동 생성 기능 통합, 템플릿 규칙 정리 및 고화질 템플릿 교체.
AI·도구
PDF 자동 생성, 템플릿 시스템, 기존 파이프라인 통합.
성과
주문 처리 흐름 안에서 표지가 자동 생성되도록 일원화, 별도 작업 단계 제거 및 산출물 품질 표준화.
전략기획 관점 — 분절된 작업을 하나의 프로세스로 통합해 핸드오프 비용 제거. → 표지 자동생성기 상세 케이스 보기
7
열람출력 제본 자동화 웹앱
문제·배경
열람출력 제본은 PDF 조회·병합 결정·워터마크·암호화 등 단계가 많고 사람의 검토가 필요한 업무.
나의 역할
product id로 PDF 조회→목록 확인→병합 결정→워터마크/암호화→다운로드를 웹 도구로 구현하되, 중요 결정 단계는 의도적으로 사람이 통제하도록 설계. 동료가 바로 쓸 수 있는 설치 패키지로 배포.
AI·도구
Streamlit 기반 웹앱, Python, PDF 가공(병합/워터마크/암호화), 사내 배포 패키징.
성과
복잡한 수작업을 단계별 도구로 정리하면서도 중요 결정은 사람이 통제, 동료도 사용 가능하도록 확산.
전략기획 관점 — 자동화와 통제의 균형(human-in-the-loop) 설계, 본인 업무를 넘어 팀 도구로 확산시킨 확장 사례.
8
운영 프로세스 재구축 및 인수인계 (어드민 교체 대응)
문제·배경
어드민 시스템 교체로 기존 주문서 프로세스를 처음부터 재정비해야 했고, 담당자 변경 대비 문서화 필요.
나의 역할
교체 환경에 맞춰 주문서 프로세스를 재구축, 후임이 그대로 따라 할 수 있는 운영 SOP 산출.
AI·도구
프로세스 설계, 업무 문서화/SOP 작성.
성과
속인화되어 있던 운영 지식을 문서로 체계화, 담당자 부재 시에도 서비스 무중단 운영 기반 마련.
전략기획 관점 — 변화관리·비즈니스 연속성 확보, 시스템 전환 리스크를 문서·프로세스 표준화로 흡수.
